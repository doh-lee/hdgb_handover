컨텐츠 업무(사이트 업데이트 건)

업무 flow

  1. 1. 디자인 가이드 & 에셋 전달 받기 (live 최소 3일 전)

    1. ⓐ live & preview 공유 날짜 및 에셋 링크 확인

    2. ⓑ 쉐어포인트에서 가이드 및 에셋 다운로드

    3. ⓒ 동영상은 vimeo 링크로 받거나, mp4파일 # FTP 업로드 후 사용 * 현재(24년5월)기준 COS만 mp4로 전달함

  2. 2. 퍼블리싱 작업 리스트 [사이트 업데이트건] 탭에 작업 내용 기입
  3. 3. [TIS] 매장 및 기획전 복사 후 기본정보 세팅
    · SIS vs Official
    SIS(더현대닷컴 샵인샵) : 적응형 페이지, iframe 이슈 | 현재(24년5월)기준 COS, AOS 브랜드만 TIS SIS매장 따로 운영중 (타 브랜드는 오피셜 미러링)
    Official : 반응형 페이지
    · 매장속성
    일반매장 : 매장정보(URL, 전시날짜 등), CategoryListing 상품 세팅
    기획전매장 : 페이지 레이아웃(컴포넌트) 세팅
    · 매장템플릿
    매장템플릿에 따라 사용할 수 있는 컴포넌트 유형이 정해져있음 (브랜드 별로 차이가 있어 숙지 필요)

    2. ⓑ [GB매장관리] 매장 전시정보 탭 - 기획전리스트에 복사한 기획전 추가 후 전시 날짜 설정

프로젝트 지원 업무(리뉴얼 or 구축)

업무 flow

  1. 1. 글로벌 디자인 분석 후 업무 범위 파악
  2. 2. 퍼블리싱 요청사항 정리 후 수행사 전달

    * 퍼블리싱 요청사항 예시

    • · 파일 경로 - 다른 브랜드, 특히 더현대닷컴과 섞이지 않도록 한 폴더 내에 css, js 폴더로 구성 (현재 해외브랜드 폴더 구조 참고)
    • · css, js 파일 내에 어느 영역의 코드인지 간단하게 주석으로 코멘트 기입
    • · inline style, !important, tag 선택자 사용 지양
    • · mediaQuery 분기사이즈: 글로벌사이트 기준, 없을 시 가능한 일정하게 작업
    • · 페이지 타입별로 컨텐츠 영역 최상단에 class 지정 (예: main, plp, pdp, about, mypage 등)
    • · 마지막 인수인계 때 사용한 라이브러리, 산출물 폴더구조, 특이사항 등 퍼블리싱 작업 내용 공유 ex) 퍼블리싱 가이드
    • · 퍼블리싱 마크업리스트 작성 요청 ex) ARKET 퍼블리스트
  3. 3. 개발 기간 → 기획 및 수행사 문의사항 대응
  4. 4. 테스트 기간

    1. · 전시영역 통합테스트 지원

      ex) 테스트관리시스템 TMS

    2. · UI퍼블리싱 결함관리

      ex) UI 오류 체크리스트

# Bitbucket & Bamboo 매뉴얼

  1. ⓐ [Bitbucket] hm-res(오피셜)이나 front-res(더현대닷컴 샵인샵) 프로젝트 open

  2. ⓑ 가져가기 요청 만들기 (Create pull request) * 원본: 본인이 만든 branch 찾기 * 대상: dev(개발), stage(스테이지), master(운영) 순으로 병합 * front-res master 의 경우 @이성택 선임님께 병합 승인 요청 → 병합 * 차이, 제출 탭에서 수정사항 확인 필수, 본인이 작업한 파일 외에 다른 파일이 있을 경우 체크 필요

  3. ex) merge 완료 화면

  4. ⓒ [Bamboo] 에서 서버 반영

    · 오피셜 hm-res 반영 방법
    hm-res-dev(개발서버), hm-res-stage(스테이지서버) 의 경우 병합 시 자동으로 서버 반영됨 (가끔 안될 경우 수동으로 Run)
    hm-res-live(운영서버) Run버튼 클릭 후 Build 완료되면 Related deployment projects 클릭해서 서버 배포
    Deploy - dsl127img deploy
    Create new release from build result 체크 → commits 눌러서 수정한 파일 확인
    Start deployment (Status가 로 바뀌면 배치 완료)
    · 샵인샵 front-res 반영 방법
    front-res-dev(개발서버), front-res-stg(스테이지서버) Run버튼 클릭 시 Build 시작
    front-res-real-live-u2l(운영서버) 반영은 @이성택 선임님께 요청
    * dev 반영 → dev 테스트 → stage 반영 → stage 테스트 → master 반영 → master 확인 (각각 단계에서 이상 없는지 페이지 확인 필수)
